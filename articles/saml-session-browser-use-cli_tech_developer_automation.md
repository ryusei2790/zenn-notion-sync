## まず何が起きているのか（超ざっくり）

SAML認証の流れは、こんなイメージです。

1. GitHub Enterprise（GHE）にアクセス
1. → 社内の認証サーバー（IdP）にリダイレクト
1. → ログイン成功
1. → 「この人OK！」という証明をGHEに返す
1. → GHEが「この人ログイン済み」としてセッションを作る
この「ログイン済み」の状態を覚えているのが **Cookie（クッキー）** です。

## 問題の正体：Cookieが消えている

SAML認証自体は通っても、セッション（Cookie）が保持されていないため、毎回「初めて来た人」扱いになり、SAMLへ飛ばされるループが発生します。Chromeのプロセスが47個も残ったのはこのためです。

## browser-use CLIが解決できる3つの理由

1. Chromeプロファイルをそのまま読み込める
1. CookieをExport / Importできる
1. セッションが切れても復元できる
